ESSENTIAL COMMANDS:

3. COMPILE APPLICATION
-----------------------
First time compilation:
  javac Chat.java

Recompile after changes:
  javac Chat.java

Clear compiled files (if needed):
  del *.class


4. RUN THE APPLICATION
------------------------
Basic run:
  java Chat

Run in background (Windows):

Run with explicit classpath:

5. CHECK JAVA VERSION
---------------------

---------------------------

  echo %OPENROUTER_API_KEY%

COMMON ISSUES & FIXES:
======================

Issue: "javac command not found"
Fix: Java not in PATH
  Solution: 
    1. Install Java JDK 11+
    2. Add to PATH: C:\Program Files\Java\jdk-11\bin
    3. Restart Command Prompt

Issue: "Exception in thread main: ClassNotFoundException"
Fix: Class not compiled or wrong directory
  Solution:
    1. cd e:\SD\LAB 8
    2. javac Chat.java
    3. java Chat

Issue: "API Key not configured"
Fix: Environment variable not set
  Solution: See "SET ENVIRONMENT VARIABLE" section above

Issue: Connection refused
Fix: Cannot reach OpenRouter API
  Solution:
    1. Check internet: ping openrouter.ai
    2. Check API key: echo %OPENROUTER_API_KEY%
    3. Visit: https://status.openrouter.ai

Issue: OutOfMemoryError
Fix: Not enough heap memory
  Solution: java -Xmx1024m Chat


PERFORMANCE TUNING:
===================

For slow responses:
  - Try faster model: openai/gpt-4o-mini
  - Increase timeout: Edit source, change timeout values

For memory issues:
  - Use -Xmx option: java -Xmx512m Chat
  - Reduce conversation history (edit NLPProcessor)

For faster startup:
  - Pre-warm JVM: Run a simple test first

For debugging:
  - Add verbose output: javac -verbose Chat.java
